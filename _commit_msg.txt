Jarvis is now part of Windows — no window, no tray, no app

Complete architectural shift: Jarvis is no longer a desktop application.
It's a system-level background process that's invisible until summoned.

Removed:
- NotifyIconHelper.cs (no system tray icon)
- GlobalHotkeyService.cs (replaced by low-level hook)
- installer/setup.iss (no Inno Setup installer)
- MainWindow tray behavior, close-to-tray, StateChanged handlers

New:
- LowLevelKeyboardHook.cs — WH_KEYBOARD_LL hook that intercepts Win+J
  BEFORE any application sees it. Works in fullscreen games, lock screen,
  and UAC prompts. Same mechanism as Discord PTT, OBS, Steam overlay.
  Also intercepts Escape to dismiss the orb.

- OrbWindow rewritten as true Win32 overlay:
  WS_EX_NOACTIVATE — never steals focus from current app
  WS_EX_TOOLWINDOW — invisible to Alt+Tab, Task View, Win+Tab
  WS_EX_LAYERED — per-pixel alpha transparency
  DWMWA_EXCLUDED_FROM_PEEK — excluded from Aero Peek
  Applied via SetWindowLong + DwmSetWindowAttribute in SourceInitialized

- App.xaml.cs rewritten — background-only mode:
  No window shown at startup. No tray icon. No taskbar entry.
  Creates OrbWindow (hidden), installs keyboard hook, starts wake word.
  The WPF Dispatcher keeps the process alive listening for events.
  --shell flag still shows MainWindow for desktop replacement mode.

- install.ps1 — system component installer (replaces .exe installer):
  Copies to C:\Program Files\Jarvis
  Creates scheduled task (starts at login, hidden, no window)
  -ShellMode replaces Explorer via registry
  -Uninstall restores Explorer, removes task, deletes files
  No shortcuts, no Start Menu, no tray — just Win+J and "Hey Jarvis"

- publish.ps1 simplified — no more Inno Setup dependency

- orb.js — handles summon before web UI is ready (queues _pendingSummon)
  Fires orb.ready after DOMContentLoaded so C# knows the overlay is live

- README.md rewritten to reflect new architecture
